[はじめに]
-------------------------------------------------------------------------
  EMWAC IMS は、NT上で動作する、フリーのメールサービスプログラムですが、
  ユーザー数にも制限なく簡単な設定で安定したメールサービスが行え大変重
  宝していました。

  しかし、不正中継に対する制限がかけられない為、SCSMFILTERというサービ
  スプログラムで、受信後のメールをチェックしフィルタリングして、不正な
  中継を防止するしか手段がなく、受信手順中に即時拒否することが出来ない
  SCSMFILTERでは、回線やメールサーバーへの負担をかけてしまいます。

  また、残念ながら、EMWAC IMS の機能拡張作業は、既に行われていないよう
  なので、あくまで個人的に、この際不正中継に対する制限がかけられるよう
  にしたSMTP受信部分を作成し公開することにしました。

  この"SPA SMTP Receiver"(SPARS.exe)は、EMWAC IMS内の SMTPRS.exe(0.83)
  と出来るだけ互換を持たせているので、サービス交換するだけで以下のよう
  な機能を追加拡張します。

　　1.中継の制限機能 
　　2.エイリアス機能
　　3.メーリングリスト機能
　　4.ロギング機能
　　5.メールフィルタ及び、ウイルス二次感染メールの抑制
　　6.SMTP 認証(AUTH)機能

  [ご注意]
  本プログラムはシェアウエアとして公開させて頂いております。
  本バージョンの試用期間はインストールし、最初の起動から３０日間です。
  解除キーを入れず３０日間を過ぎると受信機能そのものが停止します。

  但し、既にIMSを利用しSPAM被害に遭っている方で、以下の条件にあては
  まる場合は無償で解除キーを提供しますのでメールにてご連絡ください。

  a)学術系サイト  ：ACドメインで研究・教育目的であるもの(日本国内のみ)

  書式は->ヘルプの[送金について]もしくは、readme.txtをお読み下さい。

[インストール]
-------------------------------------------------------------------------
EMWAC IMSをインストールし正常に送受信が出来ていることが前提です。

１．ADMINISTRATOR権限でログインします。

２．SPARS.ZIPを解凍し、EMWAC IMSの保存フォルダに保存しサービスに追加します。
    例）
       C:>CD imsi386
       C:\imsi386>spars -install

３．メール受信サービス "IMS SMTP Receiver"を停止します。
    [コントロールパネル]
      ->[サービス]
        ->[IMS SMTP Receiver]
          ->”動作停止”＆”手動”に変更
    注意）上記サービスは削除しないで残しておく必要があります。
   +--------------------------------------------------------------------+
   |メールユーザーのローカルグループに"IMSUsers"以外を指定している場合。|
   |"HKEY_LOCAL_MACHINE\SOFTWARE\EMWAC\IMS\\MailGroup"                  |
   |                                 REG_SZ):<default "IMSUsers">       |
   |レジストリエディタで追加を行い、メールユーザーとして設定しているロー|
   |カルグループを指定します。                                          |
   +--------------------------------------------------------------------+
   |設定を後から変更した場合、"SPARS"を再起動する必要があります。       |
   +--------------------------------------------------------------------+

４．自ドメインの設定＆確認
    [コントロールパネル]
       ->[EMWAC IMS]
           ->[Misc]
             ->[Accept Mail For:] ここに運用している自ドメイン(xxx.co.jp)
                                  を設定します。

５．メール受信サービス "SPA SMTP Receiver"を開始します。
    [コントロールパネル]
      ->[サービス]
        ->[SPA SMTP Receiver]
          ->”動作開始”＆”自動”に変更

６．動作の確認の為Telenetで、以下のようにSPARSに接続して確認してみてください。

    telnet <mail server> smtp<cr>        <---- ここを入力
    ------------------------------------------
    S:220 xxx.xxxx.co.jp SPA SMTP Receiver (1.xx) Fri, 04 Feb 2000 07:17:13 +0900
    C:VRFY <username>@<domain><cr>                <---- ここを入力
    S:250 <username>@<domain> ... User ok
    または、
    S:550 <username>@<domain> ... User unknown.
    ------------------------------------------

   上記のように、VRFYコマンドにて任意の内部ユーザー名を打って、サーバーが
   "User ok"を返せばば内部ドメインユーザーとしてSPARSが認めているので受信
   可能となります。
   "User unknown"が出ていれば、ローカルグループが正しく設定されていない可
   能性があります。

[1.中継の制限機能の追加]
-------------------------------------------------------------------------
                              [SPARS 中継の制限：○中継可／×中継不可]
                 +---------------------+-----------------------------+
                 |                     |           送信先            |
                 |                     +--------------+--------------+
                 |                     | 内部ドメイン | 外部ドメイン |
                 +----+----------------+--------------+--------------+
                 | 送 | 内部ドメイン   |      ○      |      ○      |
                 | 信 +----------------+--------------+--------------+
                 | 元 | 外部ドメイン   |      ○      |      ×      |
                 +----+----------------+--------------+--------------+
1.送信元、送信先が内部ドメインであっても、ローカルグループ"IMSUsers",
  "Lists","Aliases"に属さないユーザーに対するメール中継は拒否します。
2.[Accept Mail For:]が未定義の場合、全てのメール中継を拒否します。

■ここでの内部ドメインとは、
  [コントロールパネル]->[EMWAC IMS]->[Misc]->[Accept Mail For:]
  の項目に定義されたメールサーバー内で管理されるドメイン名を示します。

■ここでの外部ドメインとは、
  [コントロールパネル]->[EMWAC IMS]->[Misc]->[Accept Mail For:]
  の項目に定義された以外の外部へ中継されるドメイン名を示します。

★特定のIPアドレス(外部ドメイン)からの中継を許可したい場合。
 SPA実行フォルダ下に用意された"effect.dat"ファイルをメモ帳などで編集し、
 特定のIPアドレスからの送信に関して許可設定します。
 （詳細は、"effect.dat"ファイルを参照下さい。）

★特定のＩＰアドレスからの受信拒否をしたい場合。
 SPA実行フォルダ下に用意された"effect.dat"ファイルをメモ帳などで編集し、
 特定のIPアドレスからの送信に関して拒否設定します。
（詳細は、"effect.dat"ファイルを参照下さい。）

★メール受信時の最大サイズを制限したい場合。
  "HKEY_LOCAL_MACHINE\SOFTWARE\EMWAC\IMS\MailInMaxSize"（DWORD) <default 0>:未制限
   （0の場合、サイズ制限は行いません。）
    注意)本レジストリは設定を行いたい場合に、レジストリエディタで追加を
         行う必要があります。
         設定を変更した場合、"SPARS"を再起動する必要があります。

★メールボックスの最大保管サイズを制限したい場合。
  "HKEY_LOCAL_MACHINE\SOFTWARE\EMWAC\IMS\MailInBoxSize"（DWORD) <default 0>:未制限
   （0の場合、サイズ制限は行いません。）
    注意)本レジストリは設定を行いたい場合に、レジストリエディタで追加を
         行う必要があります。
         なお、[MailBox Directory]の設定に"<絶対パス>\%USERNAME%"を利用
         した場合のみ有効です。
         設定を変更した場合、"SPARS"を再起動する必要があります。

★複数ドメイン使用時に特定のアカウントに特定のドメインだけを割当てたい場合。
  メールグループ(IMSUSers)を設定しただけで複数ドメインを設定すると、
  通常全てのメールアカウントが全てのドメインで受付けられてしまいます。
  
  アカウント毎に、特定のドメインのみで受付けたい場合には、

  1.ドメインユーザーマネージャに[各ドメイン名]にて[ローカルドメインを作成]する。
  2.[各カウントに作成したローカルドメインを割当て]てください。
    この際、メールグループ(IMSUSers)も一緒に設定しておいてください。

  例) 使用しているドメインが、[abc.co.jp] と [def.co.jp] であり、
      [アカウント abc-info] は、[abc.co.jp] のみで受付けたいとき。

      a)[ドメインユーザーマネージャ]にて[abc.co.jp]という名前でローカル
        グループを作成します。
      b)[ドメインユーザーマネージャ]にて[アカウント abc-info]を選択し
        [所属するグループ]へ追加設定します。
       これで、abc-info@def.co.jpでの受付は拒否されます。

[2.エイリアス機能の拡張]
-------------------------------------------------------------------------
SPARSでは、エイリアスに以下の機能拡張を行っています。

a)エイリアスの設定で複数ドメインに対応する。
  たとえば、複数ドメインの混在した環境(abc.co.jp,def.co.jp等)で運用する
  場合、両方のドメインで、info や supportのアカウントで受取れるようにし
  たい場合、  [コントロールパネル]->[EMWAC IMS]->[Aliases]の設定に

 　User name:"info@abc.co.jp" -> Map to:"abc-info@abc.co.jp"
　 User name:"info@def.co.jp" -> Map to:"def-info@def.co.jp"
 
  と設定することにより、

   "info@abc.co.jp"宛てに届いたメールは、"abc-info@abc.co.jp"へ
   "info@def.co.jp"宛てに届いたメールは、"def-info@def.co.jp"へ

  それぞれ配送されます。

  また、ワイルドカード（アスタリスク）を使う定義も可能です。

 　User name:"*@abc.co.jp" -> Map to:"abc-postmaster@abc.co.jp"

  ドメインabc.co.jp宛てのメールは"abc-postmaster@abc.co.jp"へ送られます。

   なお、SPARSでは、ドメインなしでのエイリアス設定は中継拒否されてしまいます。
   例)
     × User name:"info"           -> Map to:"abc-info"
     × User name:"info"           -> Map to:"abc-info@abc.co.jp"
     × User name:"info@abc.co.jp" -> Map to:"abc-info"
 　  ○ User name:"info@abc.co.jp" -> Map to:"abc-info@abc.co.jp"
   SPRASを導入以前に登録されているエイリアス設定には、ドメイン名も含め入力するようにして下さい。

[3.メーリングリスト機能の拡張]
-------------------------------------------------------------------------
SPARSでは、メーリングリストに以下の機能拡張を行っています。

a)メーリングリストアドレスでの受信時に"Reply-To:"ヘッダーの追加。

b)メーリングリストにタイトル及び連番を付加する設定の追加。
  タイトル及び連番を付加するには、新規のメーリングリスト作成のたびに、
  レジストリエディタで設定追加を行う必要があります。
  [メーリングリスト処理に追加した新しいレジストリ]を参照して下さい。

-------------------------------------------------------------------------
[メーリングリスト処理に追加した新しいレジストリ]
-------------------------------------------------------------------------
[1] メーリングリストカウンタ（REG_DWORD):<default 0>
    "HKEY_LOCAL_MACHINE\SOFTWARE\EMWAC\IMS\\Lists\<Mailing lists name>\MlId"
    対応するメーリングリストに投稿された回数をカウントします。

    本レジストリは、このメーリングリストにメール送信された時点で自動的に
    作成されます。(初期値 0)

    特定の番号から始めたい場合は、直接手作業でレジストリに追加して番号を
    設定してください。(その番号からカウントされます。)


[2] メーリングリスト投稿表題への名称＆カウント付加（REG_SZ):<default 空白>
    "HKEY_LOCAL_MACHINE\SOFTWARE\EMWAC\IMS\\Lists\<Mailing lists name>\MlToken"
    対応するメーリングリストに投稿された、表題(Subject)にリスト固有の名称
    及び、[1]で示すカウント数を付加します。

    カウント数の指定は、"%0<カウント桁数>d"で表現します。

    注意)本レジストリは新規のメーリングリスト作成の度に、レジストリエディタ
         で追加を行う必要があります。

    例1) "(test %03d)"
        subject: (test 001) 投稿された表題
        と置き換えられ順次（000～999)カウントアップを繰り返します。

    例2) "(test %04d)"
        subject: (test 0001) 投稿された表題
        と置き換えられ順次（0000～9999)カウントアップを繰り返します。

    例3) "(test %05d)"
        subject: (test 00001) 投稿された表題
        と置き換えられ順次（00000～99999)カウントアップを繰り返します。

[3] Reply-To:の付加の有無（REG_DWORD):<default 1>
    "HKEY_LOCAL_MACHINE\SOFTWARE\EMWAC\IMS\\Lists\<Mailing lists name>\ReplyTo"
    対応するメーリングリストへの投稿メールにReply-To:ヘッダを付加するか否かを指定します。
    付加する:1 付加しない:0

[4.ロギング機能の拡張]
-------------------------------------------------------------------------
SPARSでは、ロギングに以下の機能拡張を行っています。

a)受信メールの複写転送
  受信したメールを指定した別のアカウントに複写転送し、ログとして残します。

  "HKEY_LOCAL_MACHINE\SOFTWARE\EMWAC\IMS\MailInCopy"（BINARY) <default NULL>:複写転送しない
   （NULLの場合、複写転送しない行いません。）

    注意)本レジストリは設定を行いたい場合に、レジストリエディタで追加を
         行う必要があります。
         設定を変更した場合、"SPARS"を再起動する必要があります。
  本レジストリへの複写転送先アドレスは複数の登録が可能ですが、各アドレス
  の区切りと最後に、'0x00'(NULL)を挿入して下さい。

  例1) abc@abc.co.jp,bcd@bcd.co.jpへ複写転送するとき。
  Binary: 616263406162632E636F2E6A7000626364406263642E636F2E6A7000
           a b c @ a b c . c o . j p^^ b c d @ b c d . c o . j p^^
                                   (NULL)                      (NULL)

b)送信を試みたＩＰアドレスのログ
  SPARSに接続し、"MAIL FROM:"コマンドのまで実行をを試みたサーバー及びクライアントの全ての
  ＩＰアドレスをログとして残します。
  ログは、[Mail Spool Directry]下の"acceptlog"というフォルダ中に作成されます。
  "HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\SPARS\AcceptlogEnabled"（REG_DWORD):<default 0>
   （ 0:ログを残さない。 1:ログを残す。）

    注意)本レジストリは設定を行いたい場合に、レジストリエディタで追加を
         行う必要があります。
         設定を変更した場合、"SPARS"を再起動する必要があります。

[5.メールフィルタ及び、ウイルス二次感染メールの抑制]
-------------------------------------------------------------------------
本設定は、フィルタしたいメールの特徴を記述して一致する場合指定した処置を施します。

たとえば、ある種のウイルスは、感染したマシンから大量の、ウイルスメールを自動的に発するも
のがあります。(二次感染)
それらのウイルスメールについても特徴を記述することによりサーバー上にて受信を防ぐことが
可能になります（既にマシンに感染したウイルスを除去するものではありません）。

また、不要な広告メール、嫌がらせのメール、デマメールなどもその特徴を記述することにより
サーバー上にて受信を防ぐことが可能になります。

a)ウイルスの二次感染メールの中継抑制
　SPARS実行フォルダに用意された、"virus.dat"の設定に従い、フィルタ対象のメール
　か否かを判定し、 "virus.dat"ファイルで指定した対応を実施します。
  "virus.dat"は、メモ帳で編集可能です。

  1. メールのバックアップの有無。
     フィルタ対象のメールのバックアップは、[メール作業用フォルダ]下の"viruslog"という
     フォルダ中に作成されます。

  2. 拒否または、警告メッセージの添付

  本機能を有効にするには、以下のレジストリの追加が必要です。
  "HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\SPARS\VirusCheck"（REG_DWORD):<default 0>
   （ 0:中継抑制しない。 1:中継抑制する。）

    注意)本レジストリは設定を行いたい場合に、レジストリエディタで追加を
