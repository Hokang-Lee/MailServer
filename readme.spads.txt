【ソフトウェア種別】  SPARSと併用するフリーウエア

【ソフト説明】  SPA SMTP Reciverと併用するSMTP送信部です。

-------------------------------------------------------------------------
[はじめに]
-------------------------------------------------------------------------
  現在、EMWAC IMS用の不正中継対策として、SPARSを公開していますが、受信部
  だけでなく送信部についても作成し、公開することとしました。

  この"SPA SMTP Delivry Agent"(SPADS.exe)は、EMWAC IMS内の SMTPDS.exe
  と出来るだけ互換を保たせ、差替えるだけで元のサービス環境を変更すること
  なく簡便に移行可能になることを目的としています。

  1.SPAがSMTP サーバーとして独立したものとしても利用可能にする。
  2.受信後の中継時間の間の改善
  3.EMWAC IMSの送信部の幾つかの不具合点の改善への試み。
  4.SMTP 認証(AUTH)での配送

  [ご注意]
  本プログラムはSPARSと一緒に利用することを目的としております。
  SPARSの解除キーを入れず３０日間を過ぎるとSPADS(送信機能)も使用できなく
  なりますのでご注意下さい。

-------------------------------------------------------------------------
[使用環境]
-------------------------------------------------------------------------
Windows NT4.0以降でEMWAC IMSを利用したメールサーバーが稼動しているマシン。
本プログラムは、Intel版用のみですのでご注意下さい。

[インストール]
-------------------------------------------------------------------------
EMWAC IMSをインストールし正常に送受信が出来ていることが前提です。

(WIN2000での使用前の注意)
 以下レジストリを確認の上、値が空白の場合は、DNSサーバーのIPアドレスを
 設定する必要があります。
 "SYSTEM\\CurrentControlSet\\Services\\Tcpip\\Parameters\\NameServer"
  例)      DNS1            DNS2
     123.456.789.012 123.456.789.013
                    +--複数入力する場合は空白で区切ります。
           
１．ADMINISTRATOR権限でログインします。

２．SPADS.ZIPを解凍し、EMWAC IMSの保存フォルダに保存しサービスに追加します。
    例）
       C:>CD imsi386
       C:\imsi386>spads -install

３．メール受信サービス "IMS SMTP Delivry Agent"を停止します。
    [コントロールパネル]
      ->[サービス]
        ->[IMS SMTP Delivry Agent]
          ->”動作停止”＆”手動”に変更
    注意）上記サービスは削除しないで残しておく必要があります。

４．メール受信サービス "SPA SMTP Delivery Agent"を開始します。
    [コントロールパネル]
      ->[サービス]
        ->[SPA SMTP Delivery Agent]
          ->”動作開始”＆”自動”に変更

-------------------------------------------------------------------------
[配送時のSMTP AUTH(認証)の設定について]
-------------------------------------------------------------------------
配送時にSMTP AUTH(認証)を実施する場合には以下の各レジストリへの追加設定が
必要となります。

1.配送時にESMTPにて送信を行う設定を追加。
  (設定を行うには、以下のレジストリ値を作成する必要が有ります。)
  HKEY_LOCAL_MACHINE
  ->SYSTEM
    ->CurrentControlSet
      ->Services
        ->SPADS
          ->ESMTP（REG_DWORD):<default 0>
                  (ESMTP:0=行わない。/1=行う。)

2.配送時にもSMTP 認証(AUTH:"PLAIN LOGIN CRAM-MD5")を行える設定を追加。
  (設定を行うには、以下のレジストリ値を作成する必要が有ります。)
  HKEY_LOCAL_MACHINE
  ->SYSTEM
    ->CurrentControlSet
      ->Services
        ->SPADS
          ->SMTPAUTHMode（REG_SZ:文字列):<default 空白>
                         (SMTPAUTHMode: "PLAIN LOGIN CRAM-MD5" のいづれかを指定)
          ->SMTPAUTHId  （REG_SZ:文字列):<default 空白>
                         (SMTPAUTHId:認証時のIDを指定)
          ->SMTPAUTHPass（REG_SZ:文字列):<default 空白>
                         (SMTPAUTHId:認証時のPASSWORDを指定)

-------------------------------------------------------------------------
[配送時リトライ期間の設定について]
-------------------------------------------------------------------------
配送時リトライ期間を調整する場合には以下の各レジストリへの追加設定が
必要となります。
(設定を行うには、以下のレジストリ値を作成する必要が有ります。)
  HKEY_LOCAL_MACHINE
  ->SYSTEM
    ->CurrentControlSet
      ->Services
        ->SPADS
          ->SendMaxRetryTime"（REG_DWORD:数値):<default 8(時間)>
                              (1時間単位で時間を指定)

-------------------------------------------------------------------------
[配送時の詳細ログ設定について]
-------------------------------------------------------------------------
配送状況の詳細なログをとりたい場合には以下の各レジストリへの追加設定が
必要となります。
(ログは、"作業フォルダ\senderlog\"に記録されます。)
(設定を行うには、以下のレジストリ値を作成する必要が有ります。)
  HKEY_LOCAL_MACHINE
  ->SYSTEM
    ->CurrentControlSet
      ->Services
        ->SPADS
          ->SenderlogEnabled"（REG_DWORD:数値):<default 0:記録しない>(1:記録する)

-------------------------------------------------------------------------
[アップデートについて]
-------------------------------------------------------------------------
プログラムの差替えは、「サービス停止」させた後、プログラムを新しいものに
コピーして「サービス起動」すれば、差替えは完了します。

この際の解除キーの再入力は基本的に必要ありません。

但し、サービスの「アンインストール」をした場合は再入力の必要があります。

-------------------------------------------------------------------------
[アンインストール]
-------------------------------------------------------------------------
１．メール受信サービス "SPA SMTP Delivry Agent"を停止します。
    [コントロールパネル]
      ->[サービス]
        ->[SPA SMTP Delivery Agent]
          ->”動作停止”に変更

２．DOSプロンプトにて、SPADS.EXE の保存フォルダに移動し以下の通り実行します。
    例）
       C:>CD imsi386
       C:\imsi386>spads -remove
       C:\imsi386>SPA SMTP Delivery Agent removed.
       C:\imsi386>
    以上でアンインストールが完了しました。
